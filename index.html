PROJECT_01

Real-Time Event Intelligence Pipeline

I built an end-to-end real-time event intelligence system that ingests, classifies, and scores millions of global events to power demand forecasting for enterprise clients. The pipeline consumes raw event streams from multiple data sources through Kafka, processes them with Spark Streaming for real-time enrichment including NLP-based classification, geospatial tagging, and temporal feature extraction then serves predictions through a low-latency FastAPI layer. I trained XGBoost and PyTorch models on historical event impact data, achieving a 28% lift in forecasting accuracy over the baseline. I designed the infrastructure on AWS ECS with Terraform, set up automated CI/CD pipelines through GitHub Actions that cut deployment time by 50%, and built Prometheus and Grafana dashboards that monitor throughput, latency, and model drift across the entire system, sustaining 99.9% uptime while handling 100K+ daily API requests. The hybrid batch-and-streaming inference architecture I designed reduced infrastructure costs by 20% without sacrificing performance.

PROJECT_02

RAG System with Automated Model Operations

I built a production-grade Retrieval-Augmented Generation system over a large domain-specific document corpus, designed from the ground up for reliability and observability. I implemented a chunking and embedding pipeline that processes documents into a vector database, selecting chunking strategies and embedding models through an A/B testing framework I wrote from scratch, accelerating experimentation cycles by 30%. The retrieval layer combines semantic search with metadata filtering, and I integrated an LLM with streaming response generation behind a REST API built in FastAPI. What makes this system production-ready is the operations layer I built around it: automated drift detection that triggers model retraining when embedding quality degrades, a monitoring stack with Prometheus metrics tracking retrieval precision and generation latency, and a CI/CD pipeline that gates deployments on evaluation benchmarks. I mentored two engineers on the team through the design, focusing on testing strategies and deployment patterns that reduced production incidents.

PROJECT_03

Real-Time Feature Engineering Platform

I built a real-time feature engineering platform that computes temporal, aggregate, and cross-entity features from raw streaming event logs and serves them to ML models with sub-10ms latency. The platform uses Kafka Streams for windowed aggregations including rolling counts, sessionization, and decay-weighted metrics combined with Airflow DAGs for offline feature backfills and training dataset generation. I designed a Redis-backed online feature store that decouples feature computation from model serving, allowing multiple downstream models to consume the same features without redundant processing. This architecture reduced data latency by 35% compared to the previous batch-only approach and improved feature quality metrics by 18% through richer temporal and geospatial signal capture. I packaged the entire platform with Docker and Terraform, deployed it on Kubernetes, and wrote integration tests across 8+ services using PyTest to ensure stability through frequent releases.
